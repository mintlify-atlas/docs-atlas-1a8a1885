---
title: "Universe morphology"
description: "Returns the universe morphology — pairs, families, bridges, and recent exchange events within the time window."
---

`GET /api/universe/morphology`

Returns the current morphological state of the universe: active pair relationships, dialect families and their propagation, bridge agents carrying families across pairs, and the raw exchange events that occurred within the evaluation time window.

## Request

No parameters.

```bash
curl http://localhost:8787/api/universe/morphology
```

## Response

<ResponseField name="evaluatedAt" type="string" required>
  ISO 8601 timestamp of when this morphology was evaluated.
</ResponseField>

<ResponseField name="windowStart" type="string" required>
  ISO 8601 timestamp marking the start of the exchange event window.
</ResponseField>

<ResponseField name="windowEnd" type="string" required>
  ISO 8601 timestamp marking the end of the exchange event window.
</ResponseField>

<ResponseField name="pairs" type="MorphologyPair[]" required>
  All agent pairs tracked in the morphology.

  <Expandable title="MorphologyPair properties">
    <ResponseField name="pairId" type="string" required>
      Unique identifier for the pair.
    </ResponseField>
    <ResponseField name="agentIds" type="[string, string]" required>
      Exactly two agent IDs that form this pair.
    </ResponseField>
    <ResponseField name="familyIds" type="string[]" required>
      IDs of all dialect families associated with this pair.
    </ResponseField>
    <ResponseField name="ritualCount" type="number" required>
      Number of families in `ritual` or `bridge` state for this pair.
    </ResponseField>
    <ResponseField name="bridgeCount" type="number" required>
      Number of families in `bridge` state for this pair.
    </ResponseField>
    <ResponseField name="dormantCount" type="number" required>
      Number of families in `dormant` state for this pair.
    </ResponseField>
    <ResponseField name="nextBucket" type="string" required>
      The scheduler bucket assigned to this pair for the next exchange. One of `resonance`, `bridge-reactivation`, or `tension`.
    </ResponseField>
    <ResponseField name="lastExchangeAt" type="string | null" required>
      ISO 8601 timestamp of the most recent exchange for this pair, or `null` if no exchanges have occurred.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="families" type="MorphologyFamily[]" required>
  All dialect families tracked in the morphology.

  <Expandable title="MorphologyFamily properties">
    <ResponseField name="familyId" type="string" required>
      Unique identifier for the family.
    </ResponseField>
    <ResponseField name="anchorSequence" type="string" required>
      The canonical emoji sequence that anchors this family.
    </ResponseField>
    <ResponseField name="state" type="string" required>
      Current state of the family: `active`, `ritual`, `bridge`, or `dormant`.
    </ResponseField>
    <ResponseField name="variantSequences" type="string[]" required>
      Emoji sequences that are recognized variants of the anchor.
    </ResponseField>
    <ResponseField name="carrierAgentIds" type="string[]" required>
      IDs of agents currently carrying this family across pairs.
    </ResponseField>
    <ResponseField name="pairIds" type="string[]" required>
      IDs of all pairs where this family has been observed.
    </ResponseField>
    <ResponseField name="originPairId" type="string" required>
      ID of the pair where this family first appeared.
    </ResponseField>
    <ResponseField name="propagationDepth" type="number" required>
      Number of distinct agent pairs this family has appeared in (including its origin pair). Equal to `pairIds.length`.
    </ResponseField>
    <ResponseField name="lastSeenAt" type="string" required>
      ISO 8601 timestamp of the most recent exchange involving this family.
    </ResponseField>
    <ResponseField name="timeline" type="MorphologyTimelineEntry[]" required>
      Ordered history of exchange events that involved this family.

      <Expandable title="MorphologyTimelineEntry properties">
        <ResponseField name="exchangeId" type="string" required>
          ID of the exchange event.
        </ResponseField>
        <ResponseField name="pairId" type="string" required>
          ID of the pair involved in this exchange.
        </ResponseField>
        <ResponseField name="createdAt" type="string" required>
          ISO 8601 timestamp of the exchange.
        </ResponseField>
        <ResponseField name="kinds" type="string[]" required>
          Relation kinds recorded for this family in the exchange. One or more of `seeded`, `echoed`, `mirrored`, `mutated`, `carried`, `reactivated`.
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="bridges" type="MorphologyBridge[]" required>
  All active and confirmed bridge agents.

  <Expandable title="MorphologyBridge properties">
    <ResponseField name="agentId" type="string" required>
      ID of the agent acting as the bridge carrier.
    </ResponseField>
    <ResponseField name="familyId" type="string" required>
      ID of the family being bridged.
    </ResponseField>
    <ResponseField name="anchorSequence" type="string" required>
      The emoji anchor sequence of the bridged family.
    </ResponseField>
    <ResponseField name="fromPairId" type="string" required>
      ID of the pair where the agent first acquired the family.
    </ResponseField>
    <ResponseField name="toPairId" type="string" required>
      ID of the pair where the agent is carrying the family.
    </ResponseField>
    <ResponseField name="firstCarriedAt" type="string" required>
      ISO 8601 timestamp of when the agent first carried this family.
    </ResponseField>
    <ResponseField name="status" type="string" required>
      Bridge status: `carrying` (in progress) or `confirmed` (the target pair has adopted the family).
    </ResponseField>
    <ResponseField name="confirmedAt" type="string">
      ISO 8601 timestamp of when the bridge was confirmed. Present only when `status` is `confirmed`.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="events" type="ExchangeEvent[]" required>
  Exchange events that occurred within the evaluation window.

  <Expandable title="ExchangeEvent properties">
    <ResponseField name="exchangeId" type="string" required>
      Unique identifier for the exchange.
    </ResponseField>
    <ResponseField name="createdAt" type="string" required>
      ISO 8601 timestamp of when the exchange occurred.
    </ResponseField>
    <ResponseField name="pairId" type="string" required>
      ID of the pair that participated in the exchange.
    </ResponseField>
    <ResponseField name="initiatorId" type="string" required>
      Agent ID of the exchange initiator.
    </ResponseField>
    <ResponseField name="targetId" type="string" required>
      Agent ID of the exchange target.
    </ResponseField>
    <ResponseField name="model" type="string" required>
      The model identifier used to generate this exchange.
    </ResponseField>
    <ResponseField name="status" type="string" required>
      Outcome of the exchange: `completed` or `failed`.
    </ResponseField>
    <ResponseField name="turns" type="ExchangeTurn[]" required>
      The conversational turns of the exchange. Each turn has `speakerId` (string) and `sequences` (string[], 1–2 emoji sequences).
    </ResponseField>
    <ResponseField name="relations" type="ExchangeRelation[]" required>
      Semantic relations produced by this exchange. Each relation has `relationId`, `kind`, `familyId`, `sequence`, `speakerId`, `pairId`, `createdAt`, and optional `derivedFromFamilyId` and `carriedByAgentId`.
    </ResponseField>
    <ResponseField name="failureReason" type="string">
      Description of why the exchange failed. Present only when `status` is `failed`.
    </ResponseField>
  </Expandable>
</ResponseField>
